# 🎯 Ceremonies

## Design Review
- **When:** Before multi-file implementation or new game feature
- **Trigger:** Auto (when change involves >5 files or new game mechanic)
- **Facilitator:** Mario (picard)
- **Participants:** Mario, Sonic, Link, Yoshi (for UI-heavy changes)
- **Duration:** 15-30 minutes
- **Outcomes:** Architecture decision, task breakdown, risk assessment

## Sprint Retro
- **When:** Manual trigger
- **Trigger:** "run a retro" or end of sprint milestone
- **Facilitator:** Mario (picard)
- **Participants:** Full squad (Mario, Sonic, Link, Yoshi, Toad, Toadette, Lakitu)
- **Duration:** 30-45 minutes
- **Outcomes:** Learnings captured in agent histories, process improvements

## Revenue Review
- **When:** Monthly or when hitting revenue milestone
- **Trigger:** Manual or automated threshold
- **Facilitator:** Toad (worf)
- **Participants:** Mario, Toad, Link (for backend billing)
- **Duration:** 20-30 minutes
- **Outcomes:** Monetization strategy adjustments, A/B test plans

## Game Launch Review
- **When:** Before first Play Store release of any game
- **Trigger:** Manual ("pre-launch review")
- **Facilitator:** Mario (picard)
- **Participants:** Full squad
- **Duration:** 45-60 minutes
- **Outcomes:** Launch checklist, risk mitigation, marketing plan

## Emergency Response
- **When:** P0 issues (Play Store takedown risk, critical bug, security breach)
- **Trigger:** Auto (P0 label or security alert)
- **Facilitator:** Mario (picard) or Toad (worf) for security
- **Participants:** Relevant agents based on issue type
- **Duration:** As needed
- **Outcomes:** Immediate fix plan, rollback decision, incident report
